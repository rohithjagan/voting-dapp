# Contributing to Voting dApp

We love your input! We want to make contributing to this project as easy and transparent as possible.

## Development Process
1. Fork the repo and create your branch from `main`.
2. Run the app locally (see README).
3. Make your changes.
4. Ensure the existing tests pass (or add new ones).
5. Commit using conventional commits (`feat:`, `fix:`, `docs:`, etc.).
6. Push and open a pull request.

## Coding Standards
- **Solidity**: Follow [Solidity Style Guide](https://docs.soliditylang.org/en/latest/style-guide.html).
- **JavaScript/React**: Use ESLint config from Create React App. Prefer functional components.
- **Styling**: Use the design tokens defined in `App.js` for consistency.

## Commit Messages
Use [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `style:` formatting, missing semi colons, etc.
- `refactor:` code change that neither fixes a bug nor adds a feature
- `test:` adding missing tests
- `chore:` maintenance

## Reporting Bugs
Please include:
- MetaMask version
- Network (Amoy)
- Steps to reproduce
- Expected vs actual behaviour

## Feature Requests
Open an issue and label it `enhancement`. Provide clear use cases and possible implementations.

## Pull Requests
1. Update the README if needed.
2. Keep the PR focused (one feature/bug per PR).
3. Ensure CI passes (if set up).

## Code of Conduct
Please note we have a [Code of Conduct](CODE_OF_CONDUCT.md); follow it in all your interactions.

Thank you for contributing! 🎉
